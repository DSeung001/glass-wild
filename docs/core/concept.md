# 게임 콘셉트

## 한 문장 정의

> 여러 개의 사육장을 프리셋 기반으로 구성하고, 도트 스타일 생태계를 배경화면에서 관찰하며, 환경과 생물 관계를 관리하여 안정적인 미니 생태계를 만드는 관찰형 사육 시뮬레이션 게임.

## 플레이어 역할

플레이어는 생물을 직접 조작하는 존재가 아니라 사육 환경을 설계하고 문제를 진단하는 관리자입니다.

플레이어의 주요 판단은 다음 세 가지입니다.

1. 현재 사육장이 선택한 생물에게 적합한가?
2. 같은 사육장에 있는 생물들이 서로 안전한가?
3. 환경 변화나 행동 이상이 발생했을 때 어떤 조치가 필요한가?

## 핵심 재미

| 재미 축 | 설명 | 플레이어 행동 |
|---|---|---|
| 환경 설계 | 생물에게 맞는 온도, 습도, 바닥재와 은신처 구성 | 장비 조절, 식물과 구조물 배치 |
| 조합 판단 | 포식, 먹이 경쟁, 영역성, 과밀 위험 판단 | 생물 선택, 합사, 분리, 이동 |
| 관찰 | 생물의 이동, 섭식, 은신, 휴식과 추적 행동 감상 | 확대 관찰, 대상 선택, 배경화면 모드 사용 |
| 문제 해결 | 경고의 원인을 찾고 가장 적절한 조치 선택 | 먹이 공급, 청소, 장비 작동, 격리 |
| 다중 운영 | 세 사육장의 위험 우선순위를 판단 | 대시보드 확인, 사육장 전환, 자원 재배치 |

## 제품 정체성

이 게임은 다음 장르의 중간 지점을 목표로 합니다.

- 테라리움 꾸미기
- 생물 관찰
- 가벼운 생태 시뮬레이션
- 다중 공간 관리
- 데스크톱 배경화면 애플리케이션

정교한 현실 재현보다 플레이어가 원인과 결과를 이해할 수 있는 명확한 규칙을 우선합니다.

## 디자인 원칙

### 1. 행동이 먼저 보인다

상태창을 열기 전에 생물의 행동으로 문제를 추측할 수 있어야 합니다.

- 습도 부족: 활동 감소, 수분이 남은 구역에 머무름
- 은신처 부족: 불안정한 이동, 다른 생물 회피
- 배고픔: 먹이 탐색 빈도 증가
- 포식: 시선 고정, 접근, 추적
- 안정: 자연스러운 이동, 섭식, 휴식

### 2. 모든 실패는 설명 가능하다

문제 발생 시 다음 네 가지를 제공합니다.

- 무엇이 문제인지
- 누가 영향을 받는지
- 왜 발생했는지
- 어떤 행동으로 해결할 수 있는지

### 3. 위험한 선택을 금지하지 않는다

위험한 합사는 강하게 경고하지만 배치를 막지 않습니다. 실험과 결과 이해가 핵심 플레이이기 때문입니다.

### 4. 사고 전에 대응 시간이 있다

포식 사고나 환경 붕괴는 즉시 발생하지 않습니다. 징후, 경고, 자동 감속 순서로 대응 시간을 제공합니다.

### 5. 방치 관찰이 가능하다

일반 상태에서 환경은 천천히 변합니다. 플레이어가 계속 클릭하지 않아도 도트 생태계를 감상할 수 있어야 합니다.

### 6. 배경화면 모드는 별도 게임이 아니다

배경화면 모드는 일반 플레이의 동일한 사육장 상태를 보여주는 관찰 인터페이스입니다. 별도 저장이나 별도 생태계를 만들지 않습니다.

## MVP 가설

### 가설 A: 관리 재미

여러 사육장의 환경과 생물 조합을 안정화하는 과정이 반복 플레이할 만큼 재미있다.

### 가설 B: 관찰 재미

플레이어가 직접 조작하지 않는 시간에도 생물의 도트 애니메이션과 작은 사건을 계속 보고 싶어 한다.

### 가설 C: 정보 명료성

플레이어가 별도 공략 없이 경고 원인과 해결 방법을 이해할 수 있다.

### 가설 D: 다중 사육장 가치

사육장 간 생물 이동과 위험 우선순위 판단이 단일 사육장보다 의미 있는 선택을 만든다.

## MVP 비목표

- 현실 생태계를 정밀하게 재현하는 것
- 모든 실제 종의 사육 정보를 제공하는 것
- 수집 희귀도와 확률형 보상
- 경제 또는 상점 운영
- 경쟁 콘텐츠
- 장기간의 자동 성장
- 복잡한 번식, 유전, 질병 치료

## 톤과 표현

| 항목 | 방향 |
|---|---|
| 그래픽 | 2D 도트 스타일 |
| 분위기 | 잔잔한 관찰, 작은 긴장, 복구 가능한 사고 |
| 폭력 표현 | 포식은 추적과 부상 중심, 사실적 묘사 배제 |
| UI | 환경 진단 도구처럼 명확하되 관찰 화면은 최소화 |
| 사운드 | 작은 이동음, 물소리, 먹이 섭취음, 경고음 |

## 검토 필요 사항

- Windows 배경화면 통합을 MVP 필수 배포 조건으로 둘지 여부
- 투명 배경 창과 실제 라이브 월페이퍼 중 우선 구현 방식
- 실제 종명 사용 여부와 가상 생물명 사용 여부
- 생물 사망을 정식 버전에도 제외할지 여부
