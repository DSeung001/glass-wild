# 사육장 바닥·벽지·수역

## 목적

사육장을 면과 수역으로 나눠, 소재·식재·유영 공간을 일관되게 정의합니다. 아트·배치·환경 효과가 같은 레이어 모델을 따릅니다.

## 용어

상세 정의·치환표는 [`glossary.md`](glossary.md)를 따릅니다. 이 문서에서 쓰는 핵심만 요약합니다.

| 용어 | 정의 |
|---|---|
| 벽지 (`wall_face`) | 사육장 뒷벽·배경면. 배경화면(D-006)과 구분 |
| 바닥 (`floor` / substrate) | 육상 또는 수중 기질 레이어 |
| 수역 (`water_body`) | 논리 격자 물 셀 집합. 물고기 유영 공간 |
| 식재 오버레이 | 바닥·벽지에 겹치는 이끼·흙 포켓·부착 식물 등 |
| zone | `air`(기상부) 또는 `water`(수중부) |

## 드로우·배치 레이어 (뒤 → 앞)

1. 벽지 베이스 (`wall_face`, zone별)
2. 수역 또는 공기 공간 (`water_body` / air)
3. 바닥 기질 (`floor`, zone별)
4. 식재·이끼 오버레이
5. 구조물·장비
6. 생물

편집 시 격자는 D-003을 따르고, 관찰·배경화면 모드에서는 격자를 숨깁니다.

## 논리 셀 태그

각 논리 셀은 다음 중 하나(또는 조합)를 가집니다.

| 태그 | 의미 |
|---|---|
| `air` | 기상부 공간 |
| `water` | 수역 (유영 가능) |
| `substrate` | 기질이 깔린 바닥 셀 |

팔루다리움에서는 하부에 `water`+수중 `substrate`, 상부에 `air`+육상 `substrate`가 한 사육장에 공존할 수 있습니다.

## 식재 가능 면

| 면 ID | 설명 | 식재 예 |
|---|---|---|
| `floor_air` | 육상 바닥 | 이끼, 바닥 덮개 식물 |
| `floor_water` | 수중 바닥 | 수초, 수중 이끼 |
| `wall_air` | 기상부 벽지 | 흙 포켓, 이끼, 등반 식물 |
| `wall_water` | 수중 벽지 | 수중 이끼, 부착 식물 |

베이스(벽지·바닥 타일)와 식재 오버레이는 **자산을 분리**합니다. 한 이미지에 구워 넣지 않습니다.

## 벽지 베이스 소재 (초안)

| ID | zone | 설명 |
|---|---|---|
| `wall_cork` | air | 코르크·수목 질감 |
| `wall_soil` | air | 흙벽 |
| `wall_rock` | air / water | 암석면 (수중 변형 가능) |
| `wall_glass_edge` | air | 유리 가장자리·전면 프레임용 |
| `wall_aquatic_rock` | water | 수중 암석·부착면 |
| `wall_aquatic_plain` | water | 단순 수중 배경 (대비 낮음) |

수중 벽지는 채도·명도를 낮춰 전경 생물 가독성을 유지합니다.

## 수역

| 항목 | 규칙 |
|---|---|
| 정의 | 논리 셀의 `water` 태그 집합 |
| 수위 | 세로 셀 높이로 표현 (shallow / deep 태그 가능) |
| 유영 | `Swim` 생물은 수역 셀 **중층** 이동 허용 (`contact_mode: water`) |
| 기질 보행 | `cherry_shrimp` 등은 수중 바닥에서 `Walk` |
| 흐름 | 선택적 `flow_dir` 태그 + 짧은 도트 애니. **유체 물리 엔진 없음** (D-009) |
| 장비 | `shallow_water`는 얕은 수역을 만드는 배치 형태/장비로 취급 |

## 생물과의 연결

| 생물 예 | 주 공간 |
|---|---|
| 네리트 달팽이, 등각류, 육상 식물 | `air` + `floor_air` / `wall_air` |
| 체리새우 | `water` + `floor_water` (Walk) |
| 앤들러 구피 | `water` 체적 유영 (Swim) |

## 환경 효과 (요약)

- 수역·수분 기질: 습도 유지에 기여, 관리 실패 시 청결도 부담
- 벽면 식재: 습도·은신·활동 구역 확대, 과밀 시 시야·밀도 부담
- 상세 수치 영향은 `docs/relations/environment.md`를 따릅니다
