# Figma 설계 계획

## 목적

Godot 구현 전에 페이지별 UI, 공통 컴포넌트, 상태 표현과 핵심 사용자 흐름을 Figma에서 검증합니다. 모든 화면을 고해상도로 완성한 뒤 개발하는 방식은 피하고, 와이어프레임과 기술 검증을 병행합니다.

## Figma 파일 페이지 구조

| 페이지 | 내용 |
|---|---|
| 00 Cover | 프로젝트 개요, 버전, 관련 문서 링크 |
| 01 Foundations | 색상, 타이포그래피, 간격, 상태 언어, 아이콘 |
| 02 Components | 버튼, 카드, 경고, 수치, 탭, 툴팁 |
| 03 Dashboard | 전체 사육장 대시보드 |
| 04 Habitat | 개별 사육장 관찰 화면 |
| 05 Edit Mode | 환경·식물·장비·생물 배치 |
| 06 Creatures | 생물 목록, 필터, 상세 패널 |
| 07 Diagnosis | 환경 비교, 합사 진단, 사육장 비교 |
| 08 Emergency | 위험 경고, 이동, 임시 격리 |
| 09 Wallpaper | 배경화면 모드와 인터랙션 진입 |
| 10 Settings | 그래픽, 시간, 알림, 배경화면 설정 |
| 11 Prototype Flows | 테스트 가능한 사용자 흐름 |
| 12 Archive | 폐기·이전 디자인 |

## Foundations에서 정의할 항목

- 기본 캔버스 해상도와 최소 지원 크기
- 도트 게임 영역과 일반 UI의 스케일 규칙
- 상태: 안정, 정보, 주의, 위험, 긴급
- 텍스트 계층: 화면 제목, 패널 제목, 본문, 보조 정보
- 간격 토큰
- 모서리 반경
- 패널과 오버레이 명도 규칙
- 상태 아이콘과 관계 아이콘
- 키보드 포커스와 선택 상태

## 디자인 단계

### 1. 정보 구조

- 화면 목록 확정
- 화면별 핵심 사용자 질문 정의
- 각 정보가 항상 보일지 선택 시 보일지 구분

### 2. 저해상도 와이어프레임

- 레이아웃과 우선순위만 검증
- 실제 도트 에셋 대신 플레이스홀더 사용
- 색상과 장식 최소화

### 3. 인터랙션 프로토타입

우선 구현할 흐름:

1. 대시보드에서 위험 사육장 찾기
2. 안전한 생물 배치
3. 포식 위험 확인과 분리
4. 습도 문제 진단과 해결
5. 배경화면 모드 전환

### 4. 기술 검증

Figma 고해상도 디자인 확정 전에 Godot에서 검증:

- 투명·테두리 없는 창
- 마우스 입력 통과와 인터랙션 전환
- 도트 스프라이트 크기와 정수 배율
- 상세 패널이 관찰 영역을 가리지 않는지

### 5. 비주얼 디자인

- Foundations 적용
- 실제 도트 에셋 샘플 반영
- 상태별 명도와 대비 검증
- 긴급 화면의 과도한 점멸 방지

## 페이지별 완료 기준

| 화면 | 완료 기준 |
|---|---|
| 대시보드 | 가장 위험한 사육장을 5초 안에 식별 가능 |
| 개별 사육장 | 관찰 영역과 진단 정보가 충돌하지 않음 |
| 편집 모드 | 변경 전·후 영향을 이해할 수 있음 |
| 생물 목록 | 현재 사육장에 안전한 생물을 필터링 가능 |
| 배치 진단 | 위험 원인과 대안을 한 화면에서 이해 가능 |
| 긴급 오버레이 | 즉시 실행할 행동이 첫 시선에 보임 |
| 배경화면 | 평상시 방해가 적고 긴급 신호는 인식 가능 |

## Figma 산출물

- Foundations 스타일 페이지
- 재사용 가능한 컴포넌트와 Variants
- 핵심 화면 데스크톱 버전
- 다섯 개 프로토타입 흐름
- 상태별 주석과 규칙
- Godot 구현용 사이즈·간격 명세
- 미확정 항목 링크: `docs/decisions/README.md`

## Figma 작업 이후 개발 진입 조건

- 핵심 흐름 5개가 클릭 가능한 프로토타입으로 연결됨
- 상태 용어와 컴포넌트가 일관됨
- 대시보드와 개별 사육장 정보 우선순위 검증 완료
- 배경화면 기술 스파이크 결과가 디자인에 반영됨
- 도트 에셋 기준 크기와 표시 배율 확정

## 검토 필요

- 기본 디자인 해상도
- UI 스케일 옵션 범위
- 도트 그래픽과 UI 타이포그래피의 스타일 일치 수준
- Figma 파일을 한 파일로 유지할지 디자인 시스템을 분리할지
