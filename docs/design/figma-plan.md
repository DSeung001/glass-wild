# Figma 설계 계획

## 목적

Godot 구현 전에 페이지별 UI, 공통 컴포넌트, 상태 표현과 핵심 사용자 흐름을 Figma에서 검증합니다. 전체 화면 고해상도를 먼저 완성하지 않고, 핵심 흐름 와이어프레임과 기술 스파이크를 병행한 뒤 세로형 프로토타입 화면만 고해상도로 확정합니다. ([D-008](../decisions/D-008-figma-exit-criteria.md))

## Foundations에서 정의할 항목

- 기준 캔버스 1920×1080과 최소 지원 1280×720 브레이크포인트
- 도트 게임 영역과 일반 UI의 정수 배율·가변 패널 규칙
- 상태: 안정, 정보, 주의, 위험, 긴급
- 텍스트 계층: 화면 제목, 패널 제목, 본문, 보조 정보
- 간격 토큰
- 모서리 반경
- 패널과 오버레이 명도 규칙
- 상태 아이콘과 관계 아이콘
- 키보드 포커스와 선택 상태
- 편집 모드 격자·스냅 가이드 (관찰 모드에서는 비표시)

## 디자인 단계

### 1. 정보 구조

- 화면 목록 확정
- 화면별 핵심 사용자 질문 정의
- 각 정보가 항상 보일지 선택 시 보일지 구분

### 2. 저해상도 와이어프레임

- 레이아웃과 우선순위만 검증
- 실제 도트 에셋 대신 플레이스홀더 사용
- 색상과 장식 최소화

### 3. 인터랙션 프로토타입

우선 구현할 흐름:

1. 대시보드에서 위험 사육장 찾기
2. 안전한 생물 배치
3. 포식 위험 확인과 분리
4. 습도 문제 진단과 해결
5. 배경화면 모드 전환 (입력 통과 ↔ 인터랙션)

### 4. 기술 검증

세로형 고해상도 확정 전에 Godot에서 검증:

- 투명·테두리 없는 창 (실패 시 무테 관찰 모드 대체안)
- 마우스 입력 통과와 단축키·트레이 인터랙션 전환
- 도트 스프라이트 32·48·64와 정수 배율
- 상세 패널이 관찰 영역을 가리지 않는지

### 5. 비주얼 디자인

- Foundations 적용
- 세로형 프로토타입에 필요한 화면만 고해상도 확정
- 실제 도트 에셋 샘플 반영
- 상태별 명도와 대비 검증
- 긴급 화면의 과도한 점멸 방지
- MVP 전체 고해상도는 세로형 플레이테스트 이후 확장

## 페이지별 완료 기준

| 화면 | 완료 기준 |
|---|---|
| 대시보드 | 가장 위험한 사육장을 5초 안에 식별 가능 |
| 개별 사육장 | 관찰 영역과 진단 정보가 충돌하지 않음 |
| 편집 모드 | 격자·스냅으로 배치하고, 변경 전·후 영향을 이해할 수 있음 |
| 생물 목록 | 현재 사육장에 안전한 생물을 필터링 가능 |
| 배치 진단 | 위험 원인과 대안을 한 화면에서 이해 가능 |
| 긴급 오버레이 | 즉시 실행할 행동이 첫 시선에 보임 |
| 배경화면 | 평상시 입력 통과로 방해가 적고, 긴급 배너·강조는 인식 가능 |

- 다섯 개 프로토타입 흐름
- 상태별 주석과 규칙
- 세로형 프로토타입용 고해상도 화면
- Godot 구현용 사이즈·간격 명세
- 관련 결정: [D-001](../decisions/D-001-platform-wallpaper.md), [D-002](../decisions/D-002-resolution-sprite-size.md), [D-003](../decisions/D-003-placement-model.md), [D-006](../decisions/D-006-wallpaper-interaction.md), [D-008](../decisions/D-008-figma-exit-criteria.md)

## Godot 세로형 프로토타입 시작 조건

다음이 모두 충족되면 Godot 세로형 프로토타입에 진입합니다. ([D-008](../decisions/D-008-figma-exit-criteria.md))

- 대시보드, 개별 사육장, 편집, 배치 진단, 긴급, 배경화면 화면의 와이어프레임 완료
- 핵심 사용자 흐름 5개가 클릭 가능한 프로토타입으로 연결됨
- 상태 용어와 컴포넌트가 일관됨
- 배경화면 기술 스파이크 결과가 디자인에 반영됨
- 도트 기준 셀 크기(기본 셀)가 스파이크에서 확인됨
- 세로형 프로토타입 화면의 고해상도 디자인 완료
