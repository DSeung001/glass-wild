# 개발 진행 순서

## 원칙

전체 디자인을 완성한 뒤 구현하는 방식이 아니라, 다음 순환을 반복합니다.

```text
기획 확정
→ Figma 와이어프레임
→ 기술 스파이크
→ 디자인 보정
→ 세로형 프로토타입
→ 플레이테스트
→ 확장
```

## Phase 0. 의사결정 확정

목표: 구현 방향을 바꿀 수 있는 P0 결정을 먼저 확정합니다.

필수 결정:

- 우선 플랫폼
- 배경화면 모드의 MVP 필수 여부
- 배경화면 구현 후보
- 기본 디자인 해상도
- 도트 셀 크기
- 자유 배치와 격자 배치
- 실제 종명과 가상 생물명
- 생물 사망 처리

산출물:

- `docs/decisions/`의 상태 업데이트
- 범위 변경 시 `docs/mvp/scope.md` 수정

## Phase 1. Figma 정보 구조와 와이어프레임

목표: 구현 전에 정보 우선순위와 화면 전환을 검증합니다.

작업:

1. Foundations 초안
2. 공통 컴포넌트
3. 전체 사육장 대시보드
4. 개별 사육장 관찰
5. 편집 모드
6. 생물 목록과 배치 진단
7. 긴급 오버레이
8. 배경화면 모드
9. 핵심 프로토타입 흐름 연결

완료 조건:

- 핵심 흐름 5개가 클릭 가능
- 상태 언어와 컴포넌트가 일관됨
- 사용자 테스트에서 핵심 정보 탐색 가능

## Phase 2. 기술 스파이크

### 2A. 배경화면 스파이크

- 투명 창
- 입력 통과
- 포커스 제외
- 다중 모니터
- 화면 배율
- 장시간 성능

### 2B. 도트 애니메이션 스파이크

- 달팽이형 기준 캐릭터
- Idle, Move, Eat, Hide
- 정수 배율
- 프레임 흔들림 검사
- 실제 이동과 프레임 속도 분리

완료 조건:

- 핵심 기술 위험이 기록됨
- 실패 시 대체안이 확정됨
- Figma 디자인에 제약이 반영됨

## Phase 3. 비주얼 기준 확정

목표: 실제 게임에 사용할 최소 디자인 시스템과 에셋 규칙을 확정합니다.

작업:

- 색상과 타이포그래피
- UI 상태 컴포넌트
- 도트 셀 크기와 팔레트
- 생물 모델 시트
- 파일명과 내보내기 규칙
- 긴급 상태 표현

완료 조건:

- 달팽이형 최종 기준 스프라이트 승인
- 대시보드와 사육장 화면 고해상도 디자인 완료
- Godot 구현용 사이즈와 간격 명세 제공

## Phase 4. 세로형 프로토타입

목표: 한 사육장에서 처음부터 복구까지 전체 흐름을 완성합니다.

포함:

- 달팽이형과 등각류형
- 고습 사육장
- 습도, 청결도, 먹이
- 환경 적합도
- 안전 합사
- 먹이 탐색과 섭식
- 습도 경고와 분무 해결
- 저장
- 배경화면 표시

완료 조건:

- `docs/mvp/vertical-slice.md` 충족
- 핵심 UX 테스트 통과
- 기술 부채와 개선 목록 작성

## Phase 5. 관계 시스템 확장

목표: 위험한 합사와 복구 재미를 검증합니다.

작업:

- 귀뚜라미형과 사마귀형 추가
- 포식 관계
- 관심, 접근, 추적, 공격 예고
- 자동 감속
- 분리와 임시 격리
- 부상과 복구 요약

완료 조건:

- 사고 전에 대응 가능
- 경고와 실제 판정이 일치
- 위험 선택의 결과가 명확함

## Phase 6. 다중 사육장 확장

목표: 세 사육장 운영의 전략적 가치를 검증합니다.

작업:

- 사육장 3개
- 고습, 온대, 건조 프리셋
- 대시보드 우선순위 정렬
- 사육장 간 생물 이동
- 이동 전 양쪽 안정도 비교
- 프리셋 변경

완료 조건:

- 세 사육장을 관리하는 이유가 플레이에서 드러남
- 이동과 재배치가 의미 있는 해결책으로 사용됨

## Phase 7. MVP 콘텐츠 확장

작업:

- 개구리형과 도마뱀형
- 식물 역할군 확장
- 바닥재, 구조물, 장비와 먹이 전체 적용
- 시나리오 4개
- 필터와 추천 기능

원칙:

- 콘텐츠 수보다 규칙 명료성과 애니메이션 품질을 우선
- 실제 종 정보는 검수 후 적용

## Phase 8. 배경화면 모드 완성

작업:

- 일반 모드와 상태 공유
- UI 최소화
- 주의·긴급 표현
- 인터랙션 전환
- 다중 모니터 설정
- 성능 품질 단계
- 장시간 안정성

완료 조건:

- 다른 앱 사용을 방해하지 않음
- 관찰 재미가 있음
- 경고가 과도하게 방해하지 않음

## Phase 9. 저장, 설정과 안정화

- 저장 버전 관리
- 비정상 종료 대응
- 그래픽과 UI 스케일
- 알림 설정
- 시간 속도
- 배경화면 설정
- 성능 측정과 최적화
- 접근성 검수

## Phase 10. MVP 플레이테스트

테스트 그룹:

- 테라리움 또는 생물 관심 사용자
- 시뮬레이션 게임 사용자
- 관련 지식이 없는 사용자
- 배경화면 관찰 기능에 관심 있는 사용자

검증 지표:

- 위험 원인 이해율
- 문제 해결 성공률
- 사육장 전환 빈도
- 위험 배치 후 대응률
- 배경화면 모드 유지 시간
- 다시 플레이 의향

## 병렬 작업 가능 영역

| 트랙 A | 트랙 B | 트랙 C |
|---|---|---|
| Figma UI | 도트 에셋 탐색 | 기술 스파이크 |
| 시스템 구현 | 애니메이션 제작 | 데이터 표 작성 |
| 플레이테스트 | 에셋 보정 | 성능 측정 |

단, 다음 의존성은 지켜야 합니다.

- 기본 해상도 결정 후 최종 UI 명세
- 셀 크기 결정 후 전체 생물 에셋 제작
- 배경화면 스파이크 후 배경화면 최종 디자인
- 관계 규칙 확정 후 포식 시스템 구현

## 현재 바로 시작할 작업

1. Figma 파일 생성과 페이지 구조 설정
2. 대시보드·사육장·배치 진단 와이어프레임
3. 달팽이형 AI 모델 시트 생성
4. Godot 배경화면 기술 스파이크
5. 달팽이형 Idle·Move 애니메이션 테스트

P0~P2 의사결정은 확정됨 (`docs/decisions/`).
