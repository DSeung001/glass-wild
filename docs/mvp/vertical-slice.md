# 첫 번째 세로형 프로토타입

## 목적

전체 콘텐츠를 만들기 전에 한 사육장에서 환경 구성, 생물 행동, 문제 진단, 해결과 배경화면 표시가 끝까지 연결되는지 검증합니다.

## 포함 화면

- 전체 사육장 대시보드 최소 버전
- 사육장 상세 화면
- 생물 선택 화면
- 배치 전 진단
- 대상 상세 패널
- 경고 오버레이
- 배경화면 모드

## 포함 콘텐츠

| 분류 | 콘텐츠 |
|---|---|
| 사육장 | 고습 프리셋 1개 |
| 생물 | 달팽이형, 등각류형 |
| 식물 | 습도 유지 식물 1종 |
| 바닥재 | 수분 보유형 1종 |
| 구조물 | 작은 은신처 1종 |
| 장비 | 분무 장치 1종 |
| 먹이 | 식물성 먹이 1종 |

## 포함 시스템

- 온도와 습도
- 청결도
- 먹이 신선도
- 배고픔
- 스트레스
- 환경 적합도
- 수용량
- 안전 합사 판정
- 이동, 섭식, 은신
- 습도 경고와 해결
- 저장과 불러오기 최소 버전
- 일반 모드와 배경화면 모드 상태 공유

## 대표 플레이 흐름

```text
사육장을 연다
→ 식물과 은신처를 배치한다
→ 달팽이형과 등각류형을 선택한다
→ 환경과 합사 진단을 확인한다
→ 생물을 배치한다
→ 먹이를 놓는다
→ 생물이 먹이를 찾아간다
→ 시간이 지나 습도가 떨어진다
→ 행동 변화와 주의 경고가 발생한다
→ 분무 장치를 작동한다
→ 습도와 행동이 회복된다
→ 배경화면 모드로 전환한다
→ 동일한 생물 상태가 계속 표시된다
```

## 기술 검증 항목

- 도트 스프라이트의 기준 셀 크기
- 정수 배율과 텍스처 필터
- AnimatedSprite 기반 상태 전환
- 실제 이동과 애니메이션 속도 동기화
- 경고에서 대상 자동 선택
- 저장 후 행동 상태 복구
- 투명 창과 입력 통과
- 30분 이상 연속 실행

## UX 검증 항목

- 사용자가 생물 배치 전 적합 여부를 이해하는가
- 습도 경고의 원인과 해결책을 찾을 수 있는가
- 달팽이형과 등각류형의 행동 차이가 보이는가
- 관찰 화면의 UI가 생물을 가리지 않는가
- 배경화면 모드가 평상시 업무를 방해하지 않는가

## 완료 조건

- 대표 플레이 흐름이 중단 없이 동작
- 모든 경고가 원인, 대상과 추천 조치를 표시
- 생물 이동과 애니메이션이 시각적으로 흔들리지 않음
- 저장 후 사육장과 생물 상태가 복구됨
- 배경화면 모드 진입과 복귀가 가능
- 핵심 흐름에 치명적 사용성 문제가 없음

## 실패 시 조정 원칙

- 관리 재미가 약함: 환경 변화와 행동 피드백을 강화
- 정보가 복잡함: 수치 수를 줄이고 추천 조치를 우선 표시
- 애니메이션 제작 부담이 큼: 상태 수와 생물 수 축소
- 배경화면 기술이 불안정함: 일반 무테 관찰 모드를 대체안으로 사용
- 다중 사육장의 가치가 불명확함: 두 번째 사육장 이동 테스트를 추가

## 다음 단계 진입 조건

- 포식자와 피식자 추가

- 시나리오 모드 제작
- 최종 Figma 비주얼 디자인 반영
