# MVP 생물 역할과 데이터

## 목적

생물은 단순 수집 대상이 아니라 환경 요구, 합사 위험, 행동 차이를 만드는 게임 규칙의 핵심 엔티티입니다.

실제 종명과 정확한 사육 수치는 아직 확정하지 않습니다. MVP에서는 역할 중심의 임시 생물군으로 설계합니다.

### access 태그

| 태그 | 의미 |
|---|---|
| `floor_air` | 기상부 바닥 보행·식재 |
| `wall_air` | 기상부 벽지 |
| `floor_water` | 수중 기질 / 수역 바닥 |
| `wall_water` | 수역에 접한 벽 |
| `swim` | 수역 셀 유영 |

선택 `access_limits`: `{ "wall_air": 5 }`처럼 태그별 cm 한도. 생략·음수 = 무제한.  
측면 뷰에서 「바닥 기준 Ncm」= 하단에서 위로 N논리 셀 (논리 셀 1cm, D-010).

종은 태그를 **조합**한다. 예: 뉴트 `["floor_air","wall_air","floor_water","swim"]` + `access_limits.wall_air = 5`.

## 상태 표현 원칙

- 환경 부적합은 행동 속도 감소와 특정 구역 체류로 표현합니다.
- 배고픔은 먹이 탐색과 접근 빈도로 표현합니다.
- 스트레스는 은신, 회피, 불규칙 이동으로 표현합니다.
- 포식은 시선 고정, 접근, 추적, 공격 예고 순서로 표현합니다.
- 텍스트 패널은 행동의 원인을 설명하는 보조 수단입니다.

## 콘텐츠 제작 순서

1. 달팽이형 생물로 이동·섭식·은신 검증
2. 등각류형 생물로 군집과 청결 보조 검증
3. 사마귀형과 귀뚜라미형으로 포식 경고 검증
4. 개구리형으로 고습·수역 복합 요구 검증
5. 도마뱀형으로 건조 환경과 넓은 공간 검증
