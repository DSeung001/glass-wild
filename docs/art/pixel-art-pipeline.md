# AI 기반 도트 에셋 제작 파이프라인

## 목적

AI 이미지 생성은 콘셉트 탐색과 핵심 포즈 제작에 사용하고, 최종 게임 에셋은 일관된 스프라이트 규격과 수동 보정을 거쳐 제작합니다.

## 기본 원칙

- 프레임마다 독립적인 프롬프트 생성은 최종 애니메이션에 사용하지 않습니다.
- 하나의 기준 스프라이트를 확정한 뒤 모든 프레임을 파생합니다.
- 외곽선, 팔레트, 카메라 각도, 바닥 접점과 몸 비율을 고정합니다.
- 생성 원본, 편집 원본과 게임용 내보내기 파일을 분리합니다.
- 투명 배경과 nearest-neighbor 확대를 전제로 합니다.

## 권장 폴더 구조

```text
assets/
  source/
    prompts/
    generated/
    aseprite/
  sprites/
    creatures/
    plants/
    items/
  palettes/
  references/
```

기획 단계에서는 실제 에셋 파일을 넣지 않고 규칙만 유지합니다.

## 스타일 가이드 항목

| 항목 | MVP 권장 초안 |
|---|---|
| 시점 | 측면 또는 측면에 가까운 3/4 시점 |
| 기본 셀 | 32×32 또는 48×48 비교 후 하나로 통일 |
| 외곽선 | 1픽셀 |
| 팔레트 | 생물당 8~16색 |
| 광원 | 좌측 상단 |
| 안티앨리어싱 | 사용하지 않음 |
| 배경 | 완전 투명 |
| 바닥 접점 | 모든 프레임 동일 |
| 프레임 여백 | 동작별 동일 |
| 확대 | 정수 배율 |

## 제작 단계

### 1. 역할과 실루엣 정의

생물의 게임 역할을 먼저 확정합니다.

- 크기
- 활동 구역
- 성향
- 대표 행동
- 다른 생물과 구분되는 실루엣

### 2. 프롬프트 기록

각 생성 결과에 다음 메타데이터를 기록합니다.

- 생물 ID
- 프롬프트
- 생성 도구와 설정
- 생성 날짜
- 선택 이유
- 수정 필요 사항

### 3. 모델 시트 생성

첫 결과물은 완성 애니메이션이 아니라 다음 기준 시트입니다.

- 기본 측면 포즈
- 약한 3/4 포즈
- 몸 비율
- 팔레트 후보
- 눈·다리·더듬이 위치
- 실루엣 비교

### 4. 기준 스프라이트 확정

수동 보정 항목:

- 캔버스 크기
- 투명 픽셀
- 외곽선 두께
- 팔레트 색상
- 중심점
- 바닥 접점
- 좌우 반전 가능 여부

### 5. 핵심 포즈 제작

한 동작의 시작, 중간, 끝 포즈를 먼저 만듭니다. 이후 중간 프레임을 추가합니다.

### 6. 수동 정리

Aseprite 또는 Pixelorama에서 다음을 검사합니다.

- 프레임 간 몸 크기 변화
- 바닥 위치 흔들림
- 팔레트 이탈
- 외곽선 깜빡임
- 눈과 팔다리 위치 변형
- 루프 시작과 끝 연결

### 7. 게임용 내보내기

- 투명 PNG
- 고정된 셀 크기
- 스프라이트 시트 또는 개별 프레임
- 파일명 규칙 준수
- 원본 배율 유지

## 파일명 규칙

```text
{entity_id}_{animation}_{direction}_{frame}.png
```

예시:

```text
humid_snail_walk_right_01.png
humid_snail_walk_right_02.png
humid_snail_eat_right_01.png
```

스프라이트 시트 사용 시:

```text
humid_snail_sprite_sheet.png
humid_snail_sprite_sheet.json
```

## 프롬프트 기본 템플릿

```text
[생물 설명], 2D pixel art game sprite,
strict side view, consistent body proportions,
[32x32 또는 48x48] sprite scale,
one-pixel outline, limited color palette,
light from upper left, transparent background,
no scenery, no text, no anti-aliasing,
readable silhouette, cozy terrarium simulation game style
```

## 품질 체크리스트

- 동일 생물의 모든 프레임이 같은 종처럼 보이는가
- 실제 게임 크기에서 실루엣을 구분할 수 있는가
- 좌우 반전 시 어색한 비대칭이 없는가
- 배경화면 모드에서 작은 크기로도 행동을 읽을 수 있는가
- 투명 경계에 반투명 픽셀이 남지 않았는가
- 프레임별 중심과 바닥 접점이 고정되어 있는가

## 라이선스 기록

- 사용한 생성 도구의 상업적 이용 조건 기록
- 외부 참고 이미지의 출처와 사용 범위 기록
- 실제 종 사진을 직접 추적하거나 복제하지 않음
- 최종 에셋의 수정 과정과 원본을 보관

## 검토 필요

- 32×32와 48×48 중 기준 셀
- 생물당 최대 팔레트 수
- 생성 도구와 프롬프트 버전 관리 방식
- Aseprite를 기본 편집 도구로 확정할지
- 에셋 생성물의 저장소 포함 범위
