# 도트 애니메이션 가이드

## 목적

생물의 행동을 작은 화면에서도 읽을 수 있게 하고, 프레임 애니메이션과 실제 이동을 자연스럽게 연결합니다.

## 핵심 원칙

- 위치 이동과 스프라이트 프레임 속도를 분리합니다.
- 캐릭터 위치는 게임 프레임에 따라 부드럽게 이동합니다.
- 도트 프레임은 6~12 FPS 범위에서 재생합니다.
- 긴 이미지 크로스페이드보다 짧은 전환 클립을 사용합니다.
- 상태 변화는 행동의 시작과 종료가 보이도록 설계합니다.

## 권장 상태 구조

```text
Idle
→ IdleToMove
→ Move
→ MoveToIdle
→ Idle
```

먹이 행동 예시:

```text
Move
→ ApproachFood
→ EatStart
→ EatLoop
→ EatEnd
→ Idle
```

위험 행동 예시:

```text
Idle
→ NoticeTarget
→ Approach
→ Chase
→ AttackWindup
→ Attack
→ Recover
```

## 공통 애니메이션 목록

| 애니메이션 | 권장 프레임 | 반복 | MVP 우선도 |
|---|---:|---|---|
| Idle | 4~6 | 반복 | 필수 |
| Move | 6~8 | 반복 | 필수 |
| Eat | 4~6 | 반복 또는 1회 | 필수 |
| Hide | 3~5 | 1회 | 생물별 |
| Sleep | 2~4 | 반복 | 선택 |
| Avoid/Flee | Move 재사용 가능 | 반복 | 관계 검증 |
| Chase | 6~8 | 반복 | 포식자 필수 |
| Attack | 4~6 | 1회 | 포식자 필수 |
| Hit | 2~3 | 1회 | 사고 검증 |
| Injured | 2~4 | 반복 | 사고 검증 |

## 생물별 MVP 최소 세트

| 생물 | 애니메이션 |
|---|---|
| 달팽이형 | Idle, Move, Eat, Hide |
| 등각류형 | Idle, Move, Eat, Avoid |
| 귀뚜라미형 | Idle, Move, Eat, Flee |
| 사마귀형 | Idle, Move, Notice, Chase, Attack |
| 개구리형 | Idle, Hop, Eat, Hide |
| 도마뱀형 | Idle, Move, Rest, Chase |

세로형 프로토타입에서는 달팽이형과 등각류형만 먼저 완성합니다.

## 부드러운 연결 규칙

### 이동 시작

- 정지 프레임에서 즉시 이동 루프로 전환하지 않습니다.
- 1~2프레임의 준비 동작을 둡니다.
- 실제 위치 이동은 준비 동작 후 시작하거나 매우 낮은 속도로 시작합니다.

### 이동 종료

- 목적지 도착 전에 이동 속도를 줄입니다.
- MoveToIdle 프레임으로 자세를 안정화합니다.
- 루프 프레임 중간에서 갑자기 멈추지 않습니다.

### 방향 전환

- 대칭 캐릭터는 좌우 반전을 사용합니다.
- 비대칭 특징이 중요한 캐릭터는 방향별 시트를 제작합니다.
- 이동 방향이 바뀔 때 한 프레임 동안 정지하거나 회전 포즈를 사용합니다.

### 행동 취소

긴급 이동이나 위험 회피로 현재 행동이 취소될 수 있습니다.

- EatLoop → Alert → Flee
- Sleep → Wake → Move
- Chase → LoseTarget → Idle

모든 상태는 최소 하나의 안전한 종료 경로를 가져야 합니다.

## 픽셀 흔들림 방지

- 모든 프레임에서 바닥 접점을 동일하게 유지
- 캔버스 크기와 피벗 고정
- 프레임마다 몸 전체를 이동시키지 않고 필요한 부위만 변형
- 반 픽셀 위치 사용 금지
- 정수 배율 사용
- nearest-neighbor 텍스처 필터 사용

## 행동 가독성

작은 도트 캐릭터에서는 실제 생물의 미세한 동작보다 과장된 실루엣 변화가 필요할 수 있습니다.

- 먹이 섭취: 머리 또는 몸의 반복적인 전후 운동
- 은신: 몸이 구조물 아래로 명확히 이동
- 경계: 정지와 시선 방향 고정
- 추적: 평상시보다 빠른 이동과 목표 방향 유지
- 스트레스: 짧고 불규칙한 이동 또는 장시간 은신

## 배경화면 모드 기준

- 애니메이션 빈도를 일반 플레이보다 낮출 수 있음
- 같은 루프가 동시에 반복되지 않도록 시작 시간을 분산
- 긴급 상태가 아니면 큰 점멸과 과도한 속도 변화 금지
- CPU 사용량을 줄이기 위해 화면 밖 또는 비활성 사육장 애니메이션을 제한

## 검수 체크리스트

- Idle 시작과 종료 프레임이 자연스럽게 연결되는가
- 이동 속도와 발·몸 움직임이 어긋나지 않는가
- 1배, 2배, 4배 속도에서 읽을 수 있는가
- 좌우 반전 시 광원과 비대칭이 문제없는가
- 행동 원인이 UI 없이도 어느 정도 구분되는가
- 실제 배경화면 크기에서 지나치게 작거나 빠르지 않은가

## 검토 필요

- 기본 애니메이션 FPS
- 위치 이동 보간 방식
- 방향 전환 애니메이션 포함 여부
- 스프라이트 시트와 개별 프레임 중 저장 방식
- 배경화면 모드의 애니메이션 품질 단계
