# 사용자 흐름

## 목적

Figma 프로토타입과 Godot 구현에서 동일한 화면 전환과 판정 순서를 사용하도록 핵심 사용자 흐름을 정의합니다.

## Flow A: 첫 사육장 확인

```text
시작 화면
→ 계속하기
→ 전체 사육장 대시보드
→ 사육장 카드 선택
→ 개별 사육장 관찰
→ 생물 선택
→ 상태와 현재 행동 확인
```

완료 기준:

- 사용자가 30초 안에 가장 위험한 사육장을 찾을 수 있음
- 생물의 현재 문제와 추천 조치를 확인할 수 있음

## Flow B: 안전한 생물 배치

```text
개별 사육장
→ 가꾸기 모드
→ 팔레트에서 동물 선택
→ 논리 셀 클릭 배치 (D-010)
→ 합사·환경 진단 (필요 시)
→ 배치 완료
→ 예상 안정도와 실제 상태 확인
```

필수 분기:

- 현재 사육장에 적합
- 환경 수정 필요
- 다른 사육장이 더 적합

## Flow B2: 지형·식재 가꾸기

```text
가꾸기 모드
→ 팔레트에서 지형(벽지·땅·물) 또는 식물·이끼 선택
→ 지형: 점(클릭) 또는 사각형(드래그 채우기)
→ 식물·이끼: 클릭 (plant_surfaces 면만 허용)
→ 표시 격자로 확인
→ 관찰 모드로 복귀
```

완료 기준:

- 사용자가 논리 셀(1cm)에 스냅된 배치를 이해할 수 있음
- 이끼·식물이 잘못된 면(바닥/벽지)에 올라가지 않음

## Flow C: 위험한 생물 배치

```text
생물 선택
→ 포식 위험 표시
→ 위험 원인과 대상 확인
→ 다른 사육장 추천
→ 사용자가 위험 배치 선택
→ 강한 최종 확인
→ 배치
→ 관심·접근·추적 행동
→ 긴급 경고
→ 분리 또는 격리
```

위험 배치를 선택하는 버튼은 기본 행동보다 시각적으로 약하게 표시합니다.

## Flow D: 환경 문제 해결

```text
주의 알림
→ 문제 대상 선택
→ 원인 진단
→ 추천 조치 비교
→ 장비 또는 배치 변경
→ 예상 효과 확인
→ 실행
→ 수치와 행동의 점진적 회복
→ 경고 해결 요약
```

## Flow E: 사육장 간 이동

```text
생물 상세
→ 이동
→ 대상 사육장 비교
→ 환경·수용량·관계 판정
→ 이동 전 양쪽 안정도 예상
→ 이동
→ 출발·도착 사육장 상태 확인
```

## Flow F: 긴급 격리

```text
긴급 경고
→ 자동 감속
→ 문제 생물과 원인 강조
→ 임시 격리
→ 격리 상태 확인
→ 환경 또는 조합 수정
→ 복귀 가능 조건 확인
→ 원래 사육장 또는 다른 사육장으로 이동
```

## Flow G: 배경화면 모드

```text
대시보드 또는 사육장 화면
→ 배경화면 모드 시작
→ UI 최소화
→ 생태계 관찰
→ 주의 아이콘 발생
→ 클릭 또는 단축키
→ 인터랙션 모드
→ 일반 플레이 화면 복귀
```

## Flow H: 프리셋 변경

```text
가꾸기 모드
→ 환경 프리셋
→ 고습·온대·건조 선택
→ 현재 배치 기준 예상 변화
→ 위험 생물과 식물 표시
→ 적용 또는 취소
→ 적용 후 환경 변화 시작
```

## 화면 전환 원칙

- 관찰 화면의 카메라 위치는 상세 패널을 열어도 유지합니다.
- 경고에서 진입하면 원인 대상이 자동 선택됩니다.
- 가꾸기 모드를 종료할 때 미적용 변경을 확인합니다.
- 긴급 상황에서 불필요한 메뉴는 숨기고 즉시 행동을 우선합니다.
- 배경화면 모드와 일반 모드는 동일한 저장 상태를 사용합니다.

## Figma 프로토타입 우선순위

1. Flow A: 첫 사육장 확인
2. Flow B: 안전한 생물 배치
3. Flow C: 위험한 생물 배치와 분리
4. Flow D: 환경 문제 해결
5. Flow G: 배경화면 전환

## 사용자 테스트 질문

- 가장 위험한 사육장을 바로 찾을 수 있는가?
- 생물을 배치하기 전에 위험 이유를 이해하는가?
- 경고가 발생했을 때 무엇을 해야 하는지 아는가?
- 사육장 간 생물 이동 결과를 예상할 수 있는가?
- 배경화면 모드에서 상태 아이콘이 방해되지 않는가?
- 생물의 행동만 보고 문제를 어느 정도 추측할 수 있는가?
