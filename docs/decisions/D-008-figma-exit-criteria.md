## 결정해야 하는 질문

Godot 구현을 시작하기 전에 Figma에서 어느 수준까지 완료할지 결정합니다.

## 선택지

### A. 전체 화면 고해상도 디자인 완료 후 구현

장점:

- 구현자가 명확한 명세를 받음
- 전체 비주얼 일관성을 미리 확인 가능

위험:

- 기술 제약 발견 시 대규모 재작업
- 실제 플레이에서 불필요한 화면까지 먼저 제작
- 구현 시작이 늦어짐

### B. 핵심 흐름 와이어프레임 후 즉시 구현

장점:

- 빠른 기술 검증
- 실제 동작 기반으로 UI 개선 가능

위험:

- 임시 UI가 장기간 남을 수 있음
- 컴포넌트 일관성이 낮아질 수 있음

### C. 핵심 흐름 프로토타입 + 기술 스파이크 후 고해상도 확정

순서:

1. Foundations 초안
2. 핵심 화면 와이어프레임
3. 프로토타입 흐름 5개
4. Godot 배경화면·도트 스파이크
5. 기술 결과 반영
6. 세로형 프로토타입에 필요한 화면만 고해상도 확정

## 권장안

C를 권장합니다.

Godot 세로형 프로토타입 시작 조건:

- 대시보드, 개별 사육장, 편집, 배치 진단, 긴급, 배경화면 화면의 와이어프레임 완료
- 핵심 사용자 흐름 5개 연결
- 상태 컴포넌트와 용어 확정
- 배경화면 기술 제약 확인
- 도트 기준 셀 크기 확인
- 세로형 프로토타입 화면의 고해상도 디자인 완료

MVP 전체 고해상도 디자인은 세로형 프로토타입 테스트 이후 확장합니다.
