# 사용자 의사결정 목록

## 목적

이 폴더에는 구현 방향, 일정과 콘텐츠 범위를 바꿀 수 있어 프로젝트 책임자가 직접 확정해야 하는 항목만 둡니다.

일반적인 기획 규칙과 추천안은 다른 문서에 작성하고, 여기에는 선택지·권장안·영향과 최종 결정을 기록합니다.

## 상태 규칙

| 상태 | 의미 |
|---|---|
| Pending | 사용자 결정 필요 |
| Proposed | 권장안이 있으나 미확정 |
| Decided | 최종 결정 완료 |
| Revisit | 구현 또는 테스트 결과로 재검토 필요 |
| Superseded | 다른 결정으로 대체됨 |

## 우선순위

| 우선순위 | 의미 |
|---|---|
| P0 | Figma 또는 기술 스파이크 시작 전 필요 |
| P1 | 세로형 프로토타입 구현 전 필요 |
| P2 | MVP 콘텐츠 확장 전 필요 |

## 결정 방법

1. 각 문서의 선택지와 영향을 확인합니다.
2. `최종 결정` 섹션을 작성합니다.
3. 상태를 `Decided`로 변경합니다.
4. 결정 날짜와 이유를 기록합니다.
5. 영향을 받는 기획 문서를 함께 수정합니다.

## 결정 기록 형식

```text
상태: Decided
결정: 선택한 내용
결정일: YYYY-MM-DD
결정자: 이름 또는 역할
이유: 선택 이유
재검토 조건: 어떤 조건에서 다시 검토할지
```
