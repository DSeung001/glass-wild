# D-006 배경화면 입력과 알림 방식

- 우선순위: P0
- 상태: Pending
- 관련 문서: `docs/technical/wallpaper-spike.md`, `docs/ui/layout.md`

## 결정해야 하는 질문

1. 배경화면 모드에서 마우스 입력을 기본적으로 통과시킬 것인가?
2. 플레이어가 어떤 방식으로 인터랙션 모드에 진입하는가?
3. 주의와 긴급 알림을 어느 정도 강도로 표시하는가?

## 선택지

### 입력 방식

- 항상 입력 통과: 업무 방해가 적지만 즉시 클릭 불가
- 항상 클릭 가능: 직관적이지만 다른 앱 조작 방해 가능
- 기본 입력 통과 + 단축키 전환: 안전성과 인터랙션 균형
- 화면 모서리 또는 트레이 아이콘으로 전환: 발견성이 낮을 수 있음

### 알림 방식

- 앱 내부 아이콘만 표시
- 짧은 배너와 대상 강조
- 운영체제 알림도 사용
- 긴급 상태에서 자동 일반 화면 전환

## 권장안

- 기본 상태: 마우스 입력 통과
- 인터랙션 진입: 사용자 지정 전역 단축키 또는 트레이 메뉴
- 인터랙션 상태: 생물과 최소 UI 클릭 가능
- 종료: 빈 영역 클릭, ESC 또는 일정 시간 후 자동 복귀
- 정보·주의: 작은 비점멸 아이콘
- 위험: 문제 사육장과 대상 강조
- 긴급: 자동 감속과 짧은 배너, 일반 화면 자동 전환은 하지 않음
- 운영체제 알림: 기본 비활성, 사용자가 선택

## 판단 기준

- 다른 앱 사용을 방해하지 않는가
- 인터랙션 모드 진입 방법을 기억할 수 있는가
- 긴급 상황을 놓치지 않는가
- 장시간 실행 시 알림 피로가 없는가
- 다중 모니터에서 일관되게 동작하는가
