# D-001 우선 플랫폼과 배경화면 MVP 기준

- 우선순위: P0
- 상태: Pending
- 관련 문서: `docs/technical/wallpaper-spike.md`, `docs/mvp/scope.md`

## 결정해야 하는 질문

1. MVP의 우선 운영체제를 Windows로 한정할 것인가?
2. 배경화면 모드가 동작하지 않으면 MVP 배포를 중단할 것인가?
3. 투명 오버레이가 실패할 경우 일반 무테 관찰 모드를 허용할 것인가?

## 선택지

### A. Windows 전용 + 배경화면 필수

- 제품 정체성이 명확함
- 테스트 범위를 줄일 수 있음
- 투명 창 호환성 문제가 전체 일정을 막을 수 있음

### B. Windows 우선 + 스파이크 통과 시 배경화면 필수

- 기술 검증 결과에 따라 현실적으로 범위를 조정 가능
- 실패 시 일반 무테 관찰 모드를 대체안으로 유지
- 초기 마케팅 문구가 조건부가 됨

### C. 일반 게임 우선 + 배경화면은 후속 기능

- 핵심 시뮬레이션 개발 위험 감소
- 제품의 차별점이 MVP에서 약해짐
- 배경화면 중심 UI를 나중에 다시 설계할 수 있음

## 권장안

B를 권장합니다.

- 우선 플랫폼은 Windows
- Phase 2 기술 스파이크 통과 시 배경화면 모드를 MVP 필수로 확정
- 실패 시 일반 무테 관찰 모드를 MVP 대체안으로 사용
- macOS와 Linux는 MVP 이후 검토

## 영향

- 성능 테스트 범위
- 배포 기준과 제품 소개 문구
