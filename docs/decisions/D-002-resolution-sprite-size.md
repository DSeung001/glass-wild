# D-002 기본 해상도와 도트 셀 크기

- 우선순위: P0
- 상태: Pending

## 결정해야 하는 질문

1. Figma와 Godot의 기준 화면 해상도는 무엇인가?
2. 생물 스프라이트의 기본 셀 크기는 32×32인가, 48×48인가?
3. UI와 게임 영역의 확대 규칙은 어떻게 할 것인가?

## 선택지

### 화면 기준

- 1920×1080 기준: 일반적인 데스크톱 구성, 공간이 넉넉함
- 1600×900 기준: 낮은 해상도 대응이 쉬움
- 가변 레이아웃 우선: 기준 해상도는 두되 패널이 유동적으로 배치됨

### 생물 셀

- 32×32: 제작량과 성능 부담이 낮고 작은 생물 표현에 적합
- 48×48: 개구리와 도마뱀의 행동 표현이 쉬우나 제작량 증가
- 혼합 셀: 생물 크기에 따라 다르게 사용하되 피벗과 배율 관리가 복잡함

## 권장안

- Figma 기준: 1920×1080
- 최소 지원 레이아웃: 1280×720에서 정보 손실 없이 동작
- UI: 가변 레이아웃
- 기술 스파이크에서 32×32 달팽이형과 48×48 달팽이형을 실제 크기로 비교
- 전체 생물은 테스트 후 하나의 기본 셀로 통일
- 화면 확대는 정수 배율을 우선

## 판단 기준

- 배경화면 모드에서 행동을 읽을 수 있는가
- 개별 사육장에 생물을 여러 마리 표시할 수 있는가
- 애니메이션 제작량을 감당할 수 있는가
- Figma 패널과 관찰 영역의 균형이 유지되는가

## 영향

- 전체 Figma 명세
- 도트 에셋 제작량
- 카메라와 UI 스케일
- 사육장에 표시 가능한 생물 수
- 성능과 메모리 사용량
