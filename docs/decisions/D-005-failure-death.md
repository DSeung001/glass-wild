# D-005 사망·부상·실패 처리

- 우선순위: P1
- 상태: Pending
- 관련 문서: `docs/core/systems.md`, `docs/mvp/scope.md`

## 결정해야 하는 질문

1. 생물이 사망할 수 있는가?
2. 포식 사고의 결과를 어느 강도로 표현할 것인가?
3. 관리 실패가 게임 종료로 이어지는가?

## 선택지

### A. 사망 없음

- 건강 악화는 탈진, 부상과 자동 격리로 처리
- 편안한 관찰 분위기 유지
- 위험의 긴장감이 약해질 수 있음

### B. MVP 사망 없음, 정식 버전에서 옵션 검토

- 초기 테스트에서 복구 플레이에 집중
- 정식 버전의 톤과 사용자 반응을 보고 결정
- 저장과 윤리적 표현 부담을 늦출 수 있음

### C. 사망 포함

- 관리 실패와 포식의 의미가 강함
- 잔잔한 배경화면 게임과 충돌 가능
- 플레이어가 방치 모드를 불안하게 느낄 수 있음

## 권장안

B를 권장합니다.

MVP 처리 순서:

```text
스트레스 또는 환경 악화
→ 위험 경고
→ 탈진 또는 부상
→ 자동 일시정지
→ 임시 격리
→ 환경 수정
→ 회복
```

포식 사고는 사실적인 묘사 없이 짧은 공격 동작, 부상 상태와 분리 조치로 표현합니다.

## 실패 원칙

- 게임 오버 없음
- 실패는 복구 과제로 연결
- 경고 없이 치명적 결과가 발생하지 않음
- 배경화면 모드에서는 사고 전에 자동 감속 또는 일시정지
- 플레이어가 장시간 자리를 비웠다는 이유만으로 회복 불가능한 손실을 주지 않음

## 영향

- 게임 톤
- 포식 애니메이션
- 건강과 격리 시스템
- 배경화면 모드의 안전성
- 저장 데이터와 플레이어 감정 부담
